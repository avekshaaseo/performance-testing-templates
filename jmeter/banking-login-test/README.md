# Banking Login Performance Test

## Objective

Simulate concurrent users logging into a banking application to validate:
- response time
- scalability
- authentication stability
- TPS under peak load

---

## Tool Used

- Apache JMeter

---

## Test Scenario

| Parameter | Value |
|---|---|
| Concurrent Users | 5000 |
| Ramp-Up Time | 10 Minutes |
| Test Duration | 30 Minutes |

---

## Metrics Captured

- Average Response Time
- 95th Percentile
- Error %
- Throughput
- Server CPU Usage

---

## Best Practices

- Use parameterization
- Monitor backend systems
- Validate session handling
- Correlate dynamic values
